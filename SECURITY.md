# Beveiligingsbeleid

## Huidige status

GBO-Semantiek is een **conceptstandaard in ontwikkeling** (status: concept, versie v0.1).
Dit repository bevat documentatie en modelartefacten (Markdown, UML, JSON Schema, RDF)
en geen draaiende applicatie of dienst. Er worden geen persoonsgegevens of
geheimen in dit repository verwerkt of opgeslagen.

## Ondersteunde versies

Alleen de meest recente uitgebrachte versie van de standaard wordt onderhouden en
ontvangt eventuele correcties. Oudere versiemappen (bijvoorbeeld `v0.1/`) blijven
beschikbaar als referentie, maar worden niet actief onderhouden.

| Versie | Ondersteund |
|--------|-------------|
| Laatste release | ✅ |
| Oudere versies | ❌ |

## Een kwetsbaarheid melden

Heb je een beveiligingsprobleem of kwetsbaarheid gevonden — bijvoorbeeld in de
publicatie-tooling, de gegenereerde website of een afhankelijkheid — meld dit dan
**niet** via een openbaar issue.

Meld het in plaats daarvan op een van de volgende manieren:

- Via e-mail aan de repository-eigenaar: **arjen.brienen@ictu.nl**
- Of via GitHub **Private Vulnerability Reporting** (tabblad *Security* →
  *Report a vulnerability*), zodra dit voor het repository is ingeschakeld.

Vermeld waar mogelijk een beschrijving van het probleem, de stappen om het te
reproduceren en de mogelijke impact. We streven ernaar om binnen vijf werkdagen te
reageren met een bevestiging en een vervolgaanpak, of een verzoek om aanvullende
informatie.
